# Design Doc — DD-00004 Gestión Documental y Facturas (Manual)

## 1. Contexto y objetivo
Este módulo define la **gestión manual de documentos y facturas** en Gestiora.  
El **documento original** (PDF) es la **fuente de verdad**, y el sistema permite su carga, asociación a proveedores y gestión de metadatos sin intervención de IA en esta fase.

El objetivo es:
- Almacenar documentos de forma segura y trazable.
- Asociar cada documento a un proveedor válido.
- Gestionar metadatos de factura manualmente.
- Permitir consulta, visualización y descarga.
- Mantener integridad histórica mediante soft delete.

**Fuera de alcance (v1):**
- Extracción automática por IA.
- Indexación semántica.
- Edición del documento original.
- Versionado de archivos.

## 3. Modelo de dominio

### 3.1 Entidades

**Factura** (Aggregate Root)
- uuid
- providerId
- fileRefId (opcional)
- status: DRAFT | ACTIVO | ELIMINADO
- numeroFactura
- fechaOperacion
- fechaVencimiento
- baseImponible
- iva
- total
- createdAt
- updatedAt
- deletedAt (nullable)

**Movimiento**
- uuid
- facturaId
- concepto
- cantidad
- precio
- baseImponible
- iva
- total

**FileRef**
- storageKey
- filename
- mimeType
- sizeBytes
- checksum

---

### 3.2 Agregados
- **Factura** es el **Aggregate Root**.
- **FileRef** es un value object tecnico asociado a la Factura.
- **Movimiento** depende de Factura.

---

### 3.3 Invariantes de dominio
- Toda Factura debe estar asociada a un proveedor ACTIVO.
- Una Factura **DRAFT** puede existir sin FileRef.
- Una Factura **ACTIVO** debe tener FileRef valido.
- El documento original **no se modifica nunca**.
- Una Factura ELIMINADO:
  - no aparece en listados operativos
  - conserva el archivo y los metadatos
- Un proveedor INACTIVO no puede recibir nuevas facturas.

## 4. Casos de uso (Application Layer)

### UC-01 Subir factura (PDF)
**Input:** archivo PDF, providerId  
**Output:** invoiceId  

**Reglas:**
- Validar formato y tamaño.
- Almacenar archivo en storage.
- Crear FileRef con checksum.
- Crear Factura en estado ACTIVO.
- Auditar carga.

**Errores:**
- `INVALID_FILE_FORMAT`
- `FILE_TOO_LARGE`
- `PROVIDER_INACTIVE`

---

### UC-02 Asociar factura a proveedor
**Input:** invoiceId, providerId  
**Output:** ok  

**Reglas:**
- Proveedor debe estar ACTIVO.
- Solo permitido si factura no está eliminada.
- Auditar asociación.

---

### UC-03 Registrar metadatos de factura
**Input:** invoiceId, datos de factura  
**Output:** ok  

**Reglas:**
- Validar campos obligatorios.
- Persistir factura asociada al documento.
- Auditar acción.

---

### UC-04 Listar facturas
**Input:** filtros (proveedor, fechas, importe), paginación  
**Output:** lista de facturas  

**Reglas:**
- Excluir eliminados por defecto.
- Filtrar por proveedor.
- Orden configurable.

---

### UC-05 Consultar factura
**Input:** invoiceId  
**Output:** metadatos + fileRef  

**Reglas:**
- Verificar permisos.
- 404 si eliminado.

---

### UC-06 Descargar / visualizar documento
**Input:** invoiceId  
**Output:** stream del archivo  

**Reglas:**
- Usuario autenticado.
- Acceso controlado.
- El storage nunca es público.

---

### UC-07 Modificar metadatos
**Input:** invoiceId, campos modificables  
**Output:** ok  

**Reglas:**
- No modificar documento original.
- Solo campos de factura.
- Auditar cambios.

---

### UC-08 Eliminación lógica
**Input:** invoiceId  
**Output:** ok  

**Reglas:**
- Marcar factura como ELIMINADO.
- No borrar archivo físico.
- Mantener trazabilidad.

## 9. Decisiones de diseño (ADR-style)

### D1 — Documento como fuente de verdad
**Decisión:** el archivo original no se modifica.  
**Consecuencia:** mayor trazabilidad y confianza legal.

### D2 — Storage desacoplado
**Decisión:** backend media todo acceso.  
**Consecuencia:** mayor seguridad y control.

### D3 — Soft delete obligatorio
**Decisión:** nunca eliminar archivos físicamente en v1.  
**Consecuencia:** costes de storage, pero máximo histórico.

---

## 10. Riesgos y evolución futura

### Riesgos
- Crecimiento de storage sin política de archivo.
- Errores de permisos exponiendo documentos.
- Metadatos inconsistentes sin IA.

### Evolución futura
- Extracción automática (DD-00005).
- Versionado de documentos.
- Soporte multi-formato.
- Archivado automático.
